# v2 extraction prompt — step-by-step + format hints + self-verify

**実施日:** 2026-04-26
**Copilot バージョン:** Microsoft Copilot Web (https://copilot.microsoft.com/)
**前提コーパス:** v2 (273 facts / 8 patterns、密度 2 倍化 + vague 吹き出し)
**意図:** v1 baseline で観測した 3 つの主要弱点を直接対策する:

1. **テーブル諦め** — p01 で 8 行 × 7 列の勤怠表を Copilot が「全セル判読不能」と出力 (recall 0.483)。1 行ずつ enumerate させる指示と、諦める前に再走査させる instruction を追加。
2. **数値フォーマット崩れ** — `60:25` (時刻) を `60.25` (decimal) と誤記、`+12%` の符号脱落など。表記ルールを明示。
3. **vague 吹き出しの対象推論抜け** — p06 で R01-R25 の本文は転記できたが、各コメントが何を指しているか書かない (recall 0.772)。本文 + 矢印先要素を併記する手順を追加。

副次的に:
- subtitle / footer の取りこぼし対策で「タイトル直下とページ最下部の薄いテキストも忘れずに」を明示
- self-verify の最終パスで件数照合 (8 行 → 8 行転記したか) を強制

## PNG / PPTX 共通プロンプト (貼付用)

```
添付したドキュメントに含まれるすべてのテキストを、漏れなく原文のまま書き起こしてください。以下の手順を必ず順番に実行してください。

# Step 1 — 構造把握 (頭の中で、出力不要)
画像またはスライドを上から下、左から右に走査し、以下を数えてください:
- 表 (table) の数 / 各表の行数・列数
- グラフ・チャートの数と種類 (棒 / 円 / 折れ線 など)
- 注釈・吹き出し・赤枠・矢印の数
- ヘッダー / サブタイトル / フッターの有無 (薄い muted テキスト含む)

# Step 2 — 要素ごとに書き起こし
以下の順序で、各要素を漏らさず書き起こしてください:

1. **タイトル + サブタイトル + メタ情報**
   - 主タイトルだけでなく、その直下にある小さい muted テキスト (実施日 / バージョン / 担当者 / チケット番号 / 環境名など) も必ず書き起こす。

2. **画面・モックアップ本体**
   - 上から下、左から右の順に UI ブロックを順次書き起こす。
   - ヘッダーの右上アイコン、通知バッジの数字、タブのアクティブ状態、ドロップダウンの開閉状態、ボタンラベルなど、小さい要素も省略しない。

3. **表 (table)**
   - 表のタイトル / 列見出し / 全行を、1 行ずつ「行番号. col1 / col2 / ... / colN」の形式で書き起こす。
   - **1 行たりとも省略しない**。N 行あれば N 行すべて。
   - 文字が小さく見える場合も即座に「判読不能」とせず、まず一度よく見てから判断する。本当にどうしても読み取れない場合のみ「[判読不能]」と明示する。

4. **グラフ・チャート**
   - タイトル、X 軸ラベル、Y 軸ラベル、凡例。
   - 各データポイントの値を全て書き出す (棒 12 本なら 12 個、円グラフ 8 セグメントなら 8 個)。

5. **コード片**
   - ファイル名 / 言語、import 行、デコレータ、関数シグネチャ (型注釈含む)、本体行、try/except の例外型まで全行を書き起こす。

6. **吹き出し・赤枠・コメント注釈**
   - 番号 (例 C1, R01, A01, D1, M1) + 本文を verbatim で。
   - **さらに、それぞれの吹き出しが指している対象を「→ ◯◯ を指している」と併記**する。判定基準は: 矢印の先の要素、または配置上最も近い画面要素。
   - 略記コメント (例 「中央?」「5→4」「16→24」「#f5f5f5」「M.O.」など) はそのまま記録し、対象要素を別途明記する。

7. **組織図 / フロー図のノード**
   - ノードごとに、書かれている全属性 (氏名 / 役職 / 入社年 / 配下人数 / 部門色帯のラベルなど) を漏らさず。
   - 親子関係 (実線) と兼務関係 (点線) があれば区別して書く。

8. **フッター**
   - ページ最下部の小さい muted テキスト (チーム名 / 機密区分 / 文書バージョンなど) も必ず書く。

# Step 3 — 表記ルール
- 時刻は HH:MM (コロン区切り)。`60:25` を `60.25` のような decimal に変換しない。
- 金額は元のまま (`¥4,200`、`164M`、`365 MJPY` など)、カンマ・通貨記号を保持。
- 割合は `%` 記号を維持 (`+12%`、`-0.5pt`)。符号 (+ / −) も保持。
- 状態記号 (○ / △ / × / ✓ / ▲) はそのまま。
- カラーコード (`#3b6db8` 等) や ピクセル値 (`16px`、`16→24`) も書かれているまま。
- 部門色帯がある場合は、色 + ラベルを併記 (例: `青 (Tech)`、`緑 (Sales)`)。

# Step 4 — 厳守事項
- 要約 / 解釈 / 言い換えは禁止。書いてあるままに書く。
- 元のドキュメントにない情報を推測で追加しない。
- 1 行 / 1 セル / 1 注釈たりとも省略しない。
- PPT の場合、各スライドを `## Slide N` (N は 1〜8) の見出しで区切る。

# Step 5 — 自己確認 (出力末尾に必ず書く)
最後に `---` 区切りを入れ、以下のフォーマットで件数照合を 1 行ずつ書いてください:
- 表: 検出 X 行 / 書き起こし Y 行 (X = Y であること)
- 注釈・吹き出し: 検出 X 個 / 書き起こし Y 個
- グラフ データポイント: 検出 X 個 / 書き起こし Y 個
- ノード (組織図など): 検出 X / 書き起こし Y

X ≠ Y のズレがあれば、戻って再走査し補完してから返答する。
```

## 使い方

1. 上のコードブロック内のテキストをそのままコピー
2. Copilot Web で `extraction_test.pptx` または `pNN_*.png` をアップロード
3. プロンプトを貼付して送信
4. 返ってきた回答を [v2/](.) ディレクトリの該当 `*_response.md` の `## Output` 行以下に貼付

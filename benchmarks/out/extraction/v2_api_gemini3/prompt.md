# v1 baseline extraction prompt

**実施日:** 2026-04-24
**Copilot バージョン:** Microsoft Copilot Web (https://copilot.microsoft.com/)
**意図:** ベースライン。シンプルに「全部書き起こして」だけ。ここからハルシネーション対策や構造指示を足していく想定。

---

## PNG / PPTX 共通プロンプト (貼付用)

```
添付したドキュメントに含まれるすべてのテキストを、漏れなく書き起こしてください。

- タイトル、本文、見出し、箇条書き、表のすべてのセル、グラフの軸ラベル・凡例・数値、図や画面内に書かれたテキスト、吹き出しや矢印の注釈、ヘッダー・フッター、ページ番号、脚注、すべて含めてください。
- 要約、解釈、言い換えは禁止です。書いてあるままに書き起こしてください。
- 元のドキュメントに書かれていない情報を追加しないでください。
- 判読できない箇所は「[判読不能]」と明示してください。
- PPT の場合は各スライドを `## Slide N` の見出しで区切ってください (N は 1〜8)。
```

---

## 使い方

1. 上のコードブロック内のテキストをそのままコピー
2. Copilot Web で `extraction_test.pptx` または `p01_ui_callouts.png` などをアップロード
3. チャット入力欄に上のプロンプトを貼付して送信
4. 返ってきた回答を同ディレクトリの対応する `*_response.md` の `## Output` 行以下に貼付

## v2 以降の改良アイデア (ここに記録しておくと比較しやすい)

- v2: 「各要素タイプを順に列挙して (見出し → 本文 → 表 → 図 → 注釈)」の step-by-step 指示
- v3: PPT 用に「各スライド内では shape を左上から右下へ順に列挙」等、フォーマット固有の CoT
- v4: 「最後にチェックリスト形式で抜けがないか確認してもう一度書き出して」の self-verify
